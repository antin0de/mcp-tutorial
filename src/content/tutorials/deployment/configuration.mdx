# Docker Deployment

Docker containerization provides a consistent, portable environment for MCP servers across development, staging, and production. This guide will show you how to containerize your MCP server using Docker and Docker Compose.

## Step 1: Create Dockerfile

Create a multi-stage Dockerfile for optimal image size:

```dockerfile
# Development Dockerfile
FROM node:18-alpine AS development

WORKDIR /app

# Copy package files
COPY package*.json ./

# Install all dependencies
RUN npm install

# Copy source code
COPY . .

# Expose port for development
EXPOSE 3000

# Development command
CMD ["npm", "run", "dev"]

# Production Dockerfile
FROM node:18-alpine AS production

WORKDIR /app

# Create non-root user
RUN addgroup -g 1001 -S nodejs
RUN adduser -S mcpuser -u 1001

# Copy package files
COPY package*.json ./

# Install only production dependencies
RUN npm ci --only=production && npm cache clean --force

# Copy built application
COPY --from=development /app/dist ./dist
COPY --from=development /app/public ./public

# Create necessary directories
RUN mkdir -p /app/logs /app/config

# Copy configuration files
COPY --chown=mcpuser:nodejs ./config ./config

# Set permissions
RUN chown -R mcpuser:nodejs /app

# Switch to non-root user
USER mcpuser

# Expose port
EXPOSE 3000

# Health check
HEALTHCHECK --interval=30s --timeout=10s --start-period=5s --retries=3 \
  CMD curl -f http://localhost:3000/health || exit 1

# Start the application
CMD ["node", "dist/http-server.js"]
```

## Step 2: Create Docker Compose Configuration

### Basic docker-compose.yml

```yaml
version: '3.8'

services:
  mcp-server:
    build:
      context: .
      dockerfile: Dockerfile
      target: production
    container_name: mcp-server
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
      - NODE_OPTIONS=--max-old-space-size=512
    volumes:
      - ./logs:/app/logs
      - ./config:/app/config
      - ./data:/app/data
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:3000/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 40s
    networks:
      - mcp-network
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"

volumes:
  mcp-logs:
  mcp-config:
  mcp-data:

networks:
  mcp-network:
    driver: bridge
```

### Advanced docker-compose.yml with nginx

```yaml
version: '3.8'

services:
  nginx:
    image: nginx:alpine
    container_name: mcp-nginx
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf
      - ./ssl:/etc/nginx/ssl
    depends_on:
      - mcp-server
    restart: unless-stopped
    networks:
      - mcp-network

  mcp-server:
    build:
      context: .
      dockerfile: Dockerfile
      target: production
    container_name: mcp-server
    expose:
      - "3000"
    environment:
      - NODE_ENV=production
      - NODE_OPTIONS=--max-old-space-size=512
      - LOG_LEVEL=info
    volumes:
      - ./logs:/app/logs
      - ./config:/app/config
      - ./data:/app/data
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "http://localhost:3000/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 40s
    networks:
      - mcp-network

volumes:
  mcp-logs:
  mcp-config:
  mcp-data:

networks:
  mcp-network:
    driver: bridge
```

## Step 3: Create nginx Configuration

### nginx.conf

```nginx
events {
    worker_connections 1024;
}

http {
    upstream mcp_backend {
        server mcp-server:3000;
        keepalive 32;
    }

    server {
        listen 80;
        server_name _;

        # Security headers
        add_header X-Frame-Options "SAMEORIGIN" always;
        add_header X-Content-Type-Options "nosniff" always;
        add_header X-XSS-Protection "1; mode=block" always;
        add_header Referrer-Policy "no-referrer-when-downgrade" always;

        # Rate limiting
        limit_req_zone $binary_remote_addr zone=api:10m rate=10r/s;
        limit_req zone=api burst=20 nodelay;

        # Health check endpoint
        location /health {
            proxy_pass http://mcp_backend/health;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }

        # MCP API endpoint
        location /mcp {
            limit_req zone=api burst=20 nodelay;

            proxy_pass http://mcp_backend/mcp;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection 'upgrade';
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;

            # Timeouts
            proxy_connect_timeout 30s;
            proxy_send_timeout 30s;
            proxy_read_timeout 30s;

            # Buffer settings
            proxy_buffering on;
            proxy_buffer_size 4k;
            proxy_buffers 8 4k;
        }

        # Static files
        location / {
            proxy_pass http://mcp_backend/;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }
    }
}
```

## Step 5: Build and Run

### Build the Docker image

```bash
# Build the development image
docker build -t mcp-server:dev -f Dockerfile --target development .

# Build the production image
docker build -t mcp-server:prod -f Dockerfile --target production .

# Build with docker-compose
docker-compose build
```

### Run the containers

```bash
# Run in detached mode
docker-compose up -d

# View running containers
docker-compose ps

# View logs
docker-compose logs -f mcp-server

# Stop containers
docker-compose down
```

### Test the deployment

```bash
# Test health endpoint
curl http://localhost/health

# Test MCP API (with nginx)
curl -X POST http://localhost/mcp \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": 1,
    "method": "tools/list",
    "params": {}
  }'

# Test without nginx
curl -X POST http://localhost:3000/mcp \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": 1,
    "method": "tools/list",
```

## Step 6: Add Monitoring and Logging

### Logging Configuration

```typescript
// src/logging.ts
import winston from 'winston';
import path from 'path';

const logFormat = winston.format.combine(
  winston.format.timestamp(),
  winston.format.errors({ stack: true }),
  winston.format.json()
);

export const logger = winston.createLogger({
  level: process.env.LOG_LEVEL || 'info',
  format: logFormat,
  transports: [
    new winston.transports.File({
      filename: path.join('/app/logs', 'error.log'),
      level: 'error',
      maxsize: 5242880, // 5MB
      maxFiles: 5,
    }),
    new winston.transports.File({
      filename: path.join('/app/logs', 'combined.log'),
      maxsize: 5242880, // 5MB
      maxFiles: 5,
    }),
  ],
});

if (process.env.NODE_ENV !== 'production') {
  logger.add(new winston.transports.Console({
    format: winston.format.simple()
  }));
}
```

### Update HTTP server to include logging

```typescript
// src/http-server.ts
import { logger } from './logging';

// Add to request handling
httpServer.on("request", (req, res) => {
  if (req.url === "/health") {
    logger.info('Health check requested', { method: req.method, url: req.url });
    res.writeHead(200);
    res.json({ status: "healthy", timestamp: new Date().toISOString() });
  } else if (req.url === "/mcp") {
    logger.info('MCP request received', {
      method: req.method,
      url: req.url,
      userAgent: req.headers['user-agent'],
      contentLength: req.headers['content-length']
    });

    // Handle MCP request
    const transport = new HTTPServerTransport(req, res);
    server.connect(transport);
  } else {
    logger.warn('Unknown endpoint requested', { method: req.method, url: req.url });
    res.writeHead(404);
    res.end("Not found");
  }
});
```

## Step 7: Docker Compose for Development

Create a development-specific docker-compose file:

```yaml

services:
  mcp-server-dev:
    build:
      context: .

      target: development
    container_name: mcp-server-dev
    ports:
      - "3000:3000"
    volumes:
      - .:/app
      - /app/node_modules
      - /app/dist
    environment:
      - NODE_ENV=development
      - DEBUG=*
    command: npm run dev
    restart: unless-stopped
    networks:
      - mcp-network

  node_modules:

networks:
  mcp-network:
    driver: bridge
```

Run with development configuration:

```bash
# Use development environment
export COMPOSE_FILE=docker-compose.dev.yml

# Build and run
docker-compose up --build -d

# Live reload changes
docker-compose logs -f
```

## Common Docker Issues and Solutions

### Permission Issues

**Problem**: Can't write to mounted volumes
```bash
# Solution: Use non-root user or adjust permissions
# In Dockerfile
USER root
RUN chown -R mcpuser:nodejs /app/logs /app/config /app/data
USER mcpuser
```

### Port Conflicts

**Problem**: Port is already in use
```bash
# Solution: Change port or find process using it
lsof -ti:3000 | xargs kill -9
# or use different port in docker-compose
ports:
  - "3001:3000"
```

### Memory Issues

**Problem**: Container using too much memory
```bash
# Solution: Set memory limits in docker-compose
services:
  mcp-server:
    deploy:
      resources:
        limits:
          memory: 512M
        reservations:
          memory: 256M
```

## Docker Security Best Practices

1. **Use non-root users**: Run containers as non-root users
2. **Minimize image size**: Use smaller base images
3. **Scan images**: Regular vulnerability scanning
4. **Use secrets**: Store sensitive data in secrets, not environment variables
5. **Read-only file systems**: Mount config and logs as volumes

# VPS Deployment

Virtual Private Servers (VPS) provide a cost-effective way to host MCP servers with full control over the environment. This guide covers setting up an MCP server on a VPS with proper security, monitoring, and maintenance.

## Prerequisites

1. **VPS Provider**: DigitalOcean, Linode, Vultr, AWS EC2, or similar
2. **Domain Name**: For production access
3. **SSH Key**: For secure access
4. **Basic Linux Knowledge**: Command line navigation

### Recommended VPS Specifications

- **CPU**: 2 cores minimum
- **Memory**: 4GB minimum
- **Storage**: 50GB SSD minimum
- **OS**: Ubuntu 22.04 LTS

## Step 1: Server Setup

### Initial Server Configuration

```bash
# Connect to your VPS via SSH
ssh root@your-vps-ip

# Update system
apt update && apt upgrade -y

# Set hostname
hostnamectl set-hostname mcp-server

# Create user
adduser mcpuser

# Add to sudo group
usermod -aG sudo mcpuser

# Switch to new user
su - mcpuser
```

### Security Hardening

```bash
# Install fail2ban
sudo apt install fail2ban -y

# Configure UFW firewall
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow ssh
sudo ufw allow http
sudo ufw allow https
sudo ufw enable

# Configure fail2ban
sudo nano /etc/fail2ban/jail.local
```

Create `/etc/fail2ban/jail.local`:

```ini
[DEFAULT]
bantime = 1h
findtime = 10m
maxretry = 5
banaction = ufw

[sshd]
enabled = true
port = ssh
filter = sshd
logpath = /var/log/auth.log
maxretry = 3
bantime = 1d
```

Restart fail2ban:

```bash
sudo systemctl restart fail2ban
```

### SSH Configuration

Edit `/etc/ssh/sshd_config`:

```bash
sudo nano /etc/ssh/sshd_config
```

```
# Security improvements
Port 2222
PermitRootLogin no
PasswordAuthentication no
ChallengeResponseAuthentication no
UsePAM yes
X11Forwarding no
PrintMotd no
AcceptEnv LANG LC_*
Subsystem sftp /usr/lib/openssh/sftp-server
```

Restart SSH:

```bash
sudo systemctl restart sshd
```

### Server Implementation

```typescript
// src/http-server.ts
import { Server } from "@modelcontextprotocol/sdk/server/index.js";
import { HTTPServerTransport } from "@modelcontextprotocol/sdk/server/http.js";
import { Server as HTTPServer } from "http";
import { json } from "body-parser";
import { createLogger, format, transports } from 'winston';

const logger = createLogger({
  level: process.env.LOG_LEVEL || 'info',
  format: format.combine(
    format.timestamp(),
    format.errors({ stack: true }),
    format.json()
  ),
  transports: [
    new transports.File({ filename: '../logs/error.log', level: 'error' }),
    new transports.File({ filename: '../logs/combined.log' }),
    new transports.Console({ format: format.simple() })
  ]
});

const server = new Server(
  {
    name: "mcp-vps-server",
    version: "1.0.0",
  },
  {
    capabilities: {
      tools: {},
      resources: {},
    },
  }
);

// Tool implementation
server.setRequestHandler(require("@modelcontextprotocol/sdk/types.js").ListToolsRequestSchema, async () => {
  return {
    tools: [
      {
        name: "echo",
        description: "Echo back the input text",
        inputSchema: {
          type: "object",
          properties: {
            text: { type: "string" },
          },
          required: ["text"],
        },
      },
    ],
  };
});

server.setRequestHandler(require("@modelcontextprotocol/sdk/types.js").CallToolRequestSchema, async (request) => {
  logger.info('Tool called', { tool: request.params.name, arguments: request.params.arguments });

  if (request.params.name === "echo") {
    return {
      content: [
        {
          type: "text",
          text: request.params.arguments.text,
        },
      ],
    };
  }
  throw new Error(`Unknown tool: ${request.params.name}`);
});

// Resource implementation
server.setRequestHandler(require("@modelcontextprotocol/sdk/types.js").ListResourcesRequestSchema, async () => {
  return {
    resources: [
      {
        uri: "resource:///system-info",
        name: "system-info",
        description: "System information",
        mimeType: "application/json",
      },
    ],
  };
});

server.setRequestHandler(require("@modelcontextprotocol/sdk/types.js").ReadResourceRequestSchema, async (request) => {
  if (request.params.uri === "resource:///system-info") {
    return {
      contents: [
        {
          uri: "resource:///system-info",
          mimeType: "application/json",
          text: JSON.stringify({
            uptime: process.uptime(),
            memory: process.memoryUsage(),
            platform: process.platform,
            nodeVersion: process.version,
          }),
        },
      ],
    };
  }
  throw new Error(`Unknown resource: ${request.params.uri}`);
});

// Create HTTP server
const httpServer = new HTTPServer();

// Add middleware
httpServer.use(json());

// Health check endpoint
httpServer.on("request", (req, res) => {
  if (req.url === "/health") {
    res.writeHead(200);
    res.json({ status: "healthy", timestamp: new Date().toISOString() });
  } else if (req.url === "/mcp") {
    logger.info('MCP request received', { method: req.method });
    const transport = new HTTPServerTransport(req, res);
    server.connect(transport);
  } else {
    res.writeHead(404);
    res.end("Not found");
  }
});

// Start server
const PORT = process.env.PORT || 3000;
httpServer.listen(PORT, () => {
  logger.info(`MCP server started on port ${PORT}`);
});
```

## Step 4: Create Systemd Service

Create `/etc/systemd/system/mcp-server.service`:

```ini
[Unit]
Description=MCP Server Service
After=network.target

[Service]
User=mcpuser
WorkingDirectory=/home/mcpuser/mcp-server
Environment=NODE_ENV=production
Environment=PATH=/usr/bin:/usr/local/bin
Environment=PORT=3000

ExecStart=/usr/bin/node dist/http-server.js
Restart=always
RestartSec=10

# Security settings
NoNewPrivileges=true
PrivateTmp=true
ProtectSystem=strict
ReadWritePaths=/home/mcpuser/mcp-server/logs

# Logging
StandardOutput=journal
StandardError=journal

# Resource limits
LimitNOFILE=65536
MemoryMax=512M

[Install]
WantedBy=multi-user.target
```

Enable and start the service:

```bash
sudo systemctl daemon-reload
sudo systemctl enable mcp-server
sudo systemctl start mcp-server

# Check status
sudo systemctl status mcp-server

# View logs
sudo journalctl -u mcp-server -f
```

## Step 5: Configure nginx

Create `/etc/nginx/sites-available/mcp-server`:

```nginx
server {
    listen 80;
    listen [::]:80;
    server_name your-domain.com www.your-domain.com;

    # Security headers
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Referrer-Policy "no-referrer-when-downgrade" always;
    add_header Content-Security-Policy "default-src 'self' http: https: data: blob: 'unsafe-inline'" always;

    # Rate limiting
    limit_req_zone $binary_remote_addr zone=api:10m rate=10r/s;
    limit_req zone=api burst=20 nodelay;

    # Logging
    access_log /var/log/nginx/mcp-server.access.log;
    error_log /var/log/nginx/mcp-server.error.log;

    # Health check
    location /health {
        proxy_pass http://localhost:3000/health;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # MCP API endpoint
    location /mcp {
        limit_req zone=api burst=20 nodelay;

        proxy_pass http://localhost:3000/mcp;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # Timeouts
        proxy_connect_timeout 30s;
        proxy_send_timeout 30s;
        proxy_read_timeout 30s;

        # Buffer settings
        proxy_buffering on;
        proxy_buffer_size 4k;
        proxy_buffers 8 4k;
    }

    # Static files
    location / {
        proxy_pass http://localhost:3000/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

# Redirect HTTP to HTTPS
server {
    listen 80;
    server_name your-domain.com www.your-domain.com;
    return 301 https://$host$request_uri;
}
```

Enable the site:

```bash
sudo ln -s /etc/nginx/sites-available/mcp-server /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

## Common Issues and Solutions

### Port Already in Use

**Problem**: Port 3000 is already in use
```bash
# Find process using port
sudo lsof -ti:3000 | xargs kill -9
# Or change port in systemd and nginx configuration
```

### Memory Issues

**Problem**: Server running out of memory
```bash
# Check memory usage
free -h
ps aux --sort=-%mem | head -10

# Increase memory limit in systemd
# Edit mcp-server.service and add:
# MemoryMax=1G
```

### Nginx Configuration Issues

**Problem**: Nginx won't start
```bash
# Test configuration
sudo nginx -t

# Check error logs
sudo tail -f /var/log/nginx/error.log
```

### Certificate Renewal Issues

**Problem**: SSL certificate renewal fails
```bash
# Manually renew
sudo certbot renew --dry-run

# Check certbot status
sudo systemctl status certbot.timer
```

## Performance Optimization

### Enable BBR Congestion Control

```bash
# Enable BBR
echo "net.core.default_qdisc=fq" | sudo tee -a /etc/sysctl.conf
echo "net.ipv4.tcp_congestion_control=bbr" | sudo tee -a /etc/sysctl.conf
sudo sysctl -p
```

### Optimize Node.js

```bash
# Set environment variables in systemd service
Environment=NODE_OPTIONS=--max-old-space-size=512 --optimize-for-size
```

### Configure Kernel Parameters

```bash
# Increase file descriptors
echo "fs.file-max = 100000" | sudo tee -a /etc/sysctl.conf
echo "* soft nofile 65536" | sudo tee -a /etc/security/limits.conf
echo "* hard nofile 65536" | sudo tee -a /etc/security/limits.conf
```

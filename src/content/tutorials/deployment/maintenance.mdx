# Azure Deployment

Azure Container Apps provides a serverless container service that makes it easy to deploy and manage MCP servers with automatic scaling, built-in monitoring, and CI/CD integration. This guide will walk you through deploying an MCP server to Azure from scratch.

## Prerequisites

1. **Azure Account**: Sign up for free at [azure.com](https://azure.com)
2. **Azure CLI**: Install and configure Azure CLI
3. **GitHub Account**: For CI/CD integration
4. **Docker Hub**: For container registry

## Step 1: Create Docker Image

### Enhanced Dockerfile for Azure

```dockerfile
# Dockerfile
FROM node:18-alpine AS builder

WORKDIR /app

# Copy package files
COPY package*.json ./

# Install dependencies
RUN npm ci

# Copy source code
COPY . .

# Build the application
RUN npm run build

# Production stage
FROM node:18-alpine AS production

WORKDIR /app

# Create non-root user
RUN addgroup -g 1001 -S nodejs
RUN adduser -S mcpuser -u 1001

# Copy package files
COPY package*.json ./

# Install production dependencies
RUN npm ci --only=production && npm cache clean --force

# Copy built application
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/public ./public

# Create necessary directories
RUN mkdir -p /app/logs /app/config /app/data

# Copy configuration files
COPY --chown=mcpuser:nodejs ./config ./config

# Set permissions
RUN chown -R mcpuser:nodejs /app

# Switch to non-root user
USER mcpuser

# Expose port
EXPOSE 3000

# Health check
HEALTHCHECK --interval=30s --timeout=10s --start-period=5s --retries=3 \
  CMD curl -f http://localhost:3000/health || exit 1

# Azure-specific logging
ENV LOG_LEVEL=info
ENV NODE_ENV=production

# Start the application
CMD ["node", "dist/http-server.js"]
```

### Build and Push to Container Registry

```bash
# Build Docker image
docker build -t mcp-server:latest .

# Tag for Azure Container Registry
docker tag mcp-server:latest mcpserver.azurecr.io/mcp-server:latest

# Login to ACR (Azure Container Registry)
az acr login --name mcpserver

# Push to ACR
docker push mcpserver.azurecr.io/mcp-server:latest

# Verify push
az acr repository list --name mcpserver --output table
```

## Step 2: Create Azure Container App

### Configure Environment Variables

```yaml
# mcp-environment.yaml
location: eastus
name: mcp-environment
properties:
  kind: Container Apps
  properties:
    platformDomainSuffix:.azurecontainerapps.io
    appLogsConfiguration:
      destination: log-analytics
      logAnalyticsConfiguration:
        customerId: "${LOG_ANALYTICS_WORKSPACE_CUSTOMER_ID}"
        sharedKey: "${LOG_ANALYTICS_WORKSPACE_SHARED_KEY}"
    dapr:
      enabled: false
    monitoring:
      destinations:
        - type: log-analytics
          name: ${LOG_ANALYTICS_WORKSPACE_NAME}
    tags:
      environment: production
```

### Create Container App with Infrastructure as Code

```bash
# Create Container App Environment
az deployment group create \
  --resource-group mcp-rg \
  --template-file environment.json \
  --parameters environment.json

# Create Container App
az containerapp create \
  --name mcp-server-app \
  --resource-group mcp-rg \
  --environment mcp-environment \
  --image mcpserver.azurecr.io/mcp-server:latest \
  --ingress 'external' \
  --target-port 3000 \
  --min-replicas 1 \
  --max-replicas 10 \
  --scale-rule-type http \
  --scale-rule-http-concurrent-target 50 \
  --scale-rule-http-auth-failure-condition false \
  --cpu 0.25 \
  --memory 0.5 \
  --secrets "API_KEY=${API_KEY}" \
  --env-vars "NODE_ENV=production" "LOG_LEVEL=info" "API_URL=https://mcp-server-app.${MCP_ENVIRONMENT}.azurecontainerapps.io"
```

## Step 3: CI/CD Pipeline with GitHub Actions

### GitHub Actions Workflow

```yaml
# .github/workflows/deploy.yml
name: Deploy MCP Server to Azure

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

env:
  REGISTRY: mcpserver.azurecr.io
  IMAGE_NAME: mcp-server

jobs:
  build-and-push:
    runs-on: ubuntu-latest
    steps:
    - name: Checkout repository
      uses: actions/checkout@v4

    - name: Set up Docker Buildx
      uses: docker/setup-buildx-action@v3

    - name: Login to Azure Container Registry
      uses: azure/login@v2
      with:
        creds: ${{ secrets.AZURE_CREDENTIALS }}

    - name: Extract metadata
      id: meta
      uses: docker/metadata-action@v5
      with:
        images: ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}
        tags: |
          type=ref,event=branch
          type=ref,event=pr
          type=sha,prefix={{branch}}-
          type=raw,value=latest,enable={{is_default_branch}}

    - name: Build and push Docker image
      uses: docker/build-push-action@v5
      with:
        context: .
        push: true
        tags: ${{ steps.meta.outputs.tags }}
        labels: ${{ steps.meta.outputs.labels }}
        cache-from: type=gha
        cache-to: type=gha,mode=max

  deploy-to-azure:
    needs: build-and-push
    runs-on: ubuntu-latest
    environment: production

    steps:
    - name: Checkout repository
      uses: actions/checkout@v4

    - name: Azure login
      uses: azure/login@v2
      with:
        creds: ${{ secrets.AZURE_CREDENTIALS }}

    - name: Update Container App
      uses: azure/CLI@v1
      with:
        inlineScript: |
          az containerapp update \
            --name mcp-server-app \
            --resource-group mcp-rg \
            --image mcpserver.azurecr.io/mcp-server:${{ github.sha }}
```

### Secrets Configuration

In GitHub repository settings, add these secrets:

- `AZURE_CREDENTIALS`: Service principal credentials for Azure
- `API_KEY`: Your MCP API key

## Step 4: Monitoring and Observability

### Application Insights Configuration

```typescript
// src/monitoring.ts
import appInsights from 'applicationinsights';

export function initializeMonitoring() {
  appInsights.setup().start();

  appInsights.defaultClient.config.appName = 'MCP Server';
  appInsights.defaultClient.config.instrumentationKey = process.env.APPLICATIONINSIGHTS_CONNECTION_STRING;

  // Track HTTP requests
  appInsights.defaultClient.autoCollectRequests = true;
  appInsights.defaultClient.autoCollectPerformance = true;
  appInsights.defaultClient.autoCollectExceptions = true;
  appInsights.defaultClient.autoCollectDependencies = true;

  return appInsights;
}

export function trackMetric(name: string, value: number, properties?: { [key: string]: any }) {
  appInsights.defaultClient.trackMetric({ name, value, properties });
}

export function trackException(error: Error, properties?: { [key: string]: any }) {
  appInsights.defaultClient.trackException({ exception: error, properties });
}
```

### Enhanced HTTP Server with Monitoring

```typescript
// src/http-server.ts
import { trackMetric, trackException } from './monitoring';

// Initialize monitoring
initializeMonitoring();

// Update request handling with tracking
httpServer.on("request", (req, res) => {
  const startTime = Date.now();

  // Track request
  trackMetric('http.request', 1, {
    method: req.method,
    url: req.url,
    userAgent: req.headers['user-agent']
  });

  // Response tracking
  res.on('finish', () => {
    const duration = Date.now() - startTime;
    trackMetric('http.response.time', duration, {
      method: req.method,
      url: req.url,
      statusCode: res.statusCode
    });
  });

  if (req.url === "/health") {
    res.writeHead(200);
    res.json({ status: "healthy", timestamp: new Date().toISOString() });
  } else if (req.url === "/mcp") {
    const transport = new HTTPServerTransport(req, res);
    server.connect(transport);
  } else {
    res.writeHead(404);
    res.end("Not found");
  }
});

// Error handling
process.on('uncaughtException', (error) => {
  trackException(error, { type: 'uncaughtException' });
  process.exit(1);
});

process.on('unhandledRejection', (reason, promise) => {
  trackException(new Error('Unhandled Rejection'), {
    reason: reason,
    promise: promise.toString()
  });
});
```

```bash
# load-test.js
import http from 'k6/http';
import { check, sleep } from 'k6';

export let options = {
  stages: [
    { duration: '30s', target: 10 },
    { duration: '1m', target: 50 },

    { duration: '1m', target: 50 },

  const payload = JSON.stringify({
    jsonrpc: "2.0",
    id: Date.now(),
    method: "tools/list",
    params: {}
```

## Troubleshooting Azure Deployment

### Common Issues and Solutions

**Problem**: Container app won't start
```bash
# Check logs
az containerapp logs --name mcp-server-app --resource-group mcp-rg --follow

# Check deployment status
az containerapp revision list --name mcp-server-app --resource-group mcp-rg --output table
```

**Problem**: Scaling not working
```bash
# Check scaling configuration
az containerapp revision list --name mcp-server-app --resource-group mcp-rg --query "[].{name:name,replicas:replicas}" --output table

# Monitor metrics
az monitor metrics list \
  --resource-group mcp-rg \
  --resource-type Microsoft.App/containerApps \
  --name "ConcurrentRequests" \
  --interval PT1M
```

**Problem**: Connection issues
```bash
# Test connectivity
az containerapp revision list --name mcp-server-app --resource-group mcp-rg --output table

# Check DNS
dig mcp-server-app.azurecontainerapps.io
```

### Azure Diagnostics


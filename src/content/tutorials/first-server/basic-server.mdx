# Section 2: Basic MCP Server

In this section, we'll create the foundation of our MCP server - the basic structure that can communicate using the MCP protocol via stdio transport.

## Understanding MCP Protocol

The MCP (Model Context Protocol) uses JSON-RPC 2.0 for communication between the server and client. The protocol works as follows:

1. **Client sends requests** to the server
2. **Server processes requests** and sends responses
3. **Communication happens via stdio** (stdin/stdout) for local development

The most important messages in MCP are:

- `initialize` - Server capabilities and configuration
- `tools/list` - List available tools
- `tools/call` - Execute a specific tool
- `notifications/list` - List available notifications

// src/index.ts
import {
  StdioServerTransport,
  JsonRpc,

## Step 2: Create Server Class

We'll create a basic server class to handle MCP messages:

```typescript
// src/index.ts
class BasicMCPServer {
  private rpc: JsonRpc;

  constructor() {
    this.rpc = new JsonRpc({
      handler: this.handleRequest.bind(this),
    });
  }

  async handleRequest(request: ServerRequest): Promise<ServerResponse> {
    const { method, params } = request;

    switch (method) {
      case 'initialize':
        return this.handleInitialize(params);
      case 'tools/list':
        return this.handleToolsList(params);
      case 'tools/call':
        return this.handleToolCall(params);
      default:
        throw new Error(`Unknown method: ${method}`);
    }
  }

  private handleInitialize(params: any) {
    return {
      capabilities: {
        tools: {},
      },
      serverInfo: {
        name: 'basic-mcp-server',
        version: '1.0.0',
      },
    };
  }

  private handleToolsList(params: any) {
    return {
      tools: [],
    };
  }

  private handleToolCall(params: any) {
    throw new Error('No tools implemented yet');
  }
}
```

## Step 3: Set Up Stdio Transport

Now, let's set up the stdio transport and start listening for messages:

```typescript
// src/index.ts
async function main() {
  const server = new BasicMCPServer();
  const transport = new StdioServerTransport();

  // Connect the RPC server to the transport
  transport.connect(server.rpc);

  console.log('MCP Server running via stdio');
}

main().catch(console.error);
```

## Understanding the Flow

Here's what happens when you run the server:

1. **Server starts** and creates a JsonRpc instance
2. **Transport connects** to stdio (stdin/stdout)
3. **Client sends messages** via stdin
4. **Server processes** messages and responds via stdout
5. **Connection continues** until process exits
